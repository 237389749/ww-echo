# ww-echo

一个基于图像识别的鸣潮声骸强化自动化工具，从 [ok-ww](https://github.com/ok-oldking/ok-wuthering-waves) 剥离，仅保留声骸相关功能，基于 [ok-script](https://github.com/ok-oldking/ok-script) 开发。

*通过 Windows 接口模拟用户进行操作，无内存读取、无文件修改*

## 功能

### 批量强化声骸 (EnhanceEchoTask)

自动将背包中的 0 级声骸强化至满级并调谐，根据设定的规则判断保留或丢弃。

**使用步骤：**
1. 游戏内按 `B` 打开背包，切换到声骸标签页
2. 使用过滤器筛选需要强化的声骸（按等级从低到高排序）
3. 选择强化策略和套装，启动任务

---

### 强化策略

提供 **传统** 和 **渐进式** 两种模式：

#### 传统模式 (Traditional)

满级后一次判断。规则：

| 规则 | 默认 | 说明 |
|------|------|------|
| 必须有双爆 | 开启 | 最终必须同时拥有暴击和暴击伤害 |
| 双爆前全有效词条 | 开启 | 双爆出现前的词条都必须在有效列表中 |
| 首条双爆 ≥ | 6.9 | 第一条双爆的数值阈值（爆伤÷2） |
| 双爆总计 ≥ | 13.8 | 暴击 + (暴击伤害÷2) ≥ 此值 |
| 有效词条 ≥ | 3 | 满级时需要的有效词条数量 |
| 第一条必为有效词条 | 开启 | 首词条必须在有效列表中 |

#### 渐进式模式 (Progressive) — 默认

每开一个词条就评估，不达标立即丢弃，**避免浪费材料**：

| 阶段 | 条件 | 逻辑 |
|:--:|------|------|
| Lv5 第1条 | 必须在套装预期词条中 | 不达标 → 直接丢弃 ✅ 节省材料 |
| Lv10 第2条 | 不做判断 | 继续强化 |
| Lv15 第3条 | 累积得分 ≥ **1.5** | 不达标 → 丢弃止损 |
| Lv20 第4条 | 累积得分 ≥ **2.25** | 不达标 → 丢弃止损 |
| Lv25 第5条 | 累积得分 ≥ **3.75** | 不达标 → 丢弃；达标 → 上锁 |

---

### 评分系统

采用 **均值归一化** 评分：每个词条得分 = `实际档位值 / 该词条全部档位均值 × 权重`，无效词条贡献 0。

```
暴击值 6.3%  →  最低档  →  6.3/8.4  = 0.75 词条当量
暴击值 10.5% →  最高档  →  10.5/8.4 = 1.25 词条当量

5个全最低 = 3.75  |  5个全平均 = 5.0  |  5个全最高 = 6.25
```

**词条权重** 在 GUI 中独立调整，共 13 个滑块，全部默认 1.0。

---

### 套装模板

17 个套装的预期词条硬编码在 `src/echo_set_templates.py`，渐进式模式的第一条必须在此列表中。

| 配置 | 默认 | 说明 |
|------|------|------|
| `当前套装` | 通用 | 17 套装 + 通用可选 |
| `有效词条` | 暴击/爆伤/攻击%/攻击/共鸣效率 | 通用默认（多选） |

---

### 批量修改声骸主属性 (ChangeEchoTask)

自动将背包中声骸的主属性修改为目标属性（数据重构）。

**目标属性选项：** 攻击、暴击伤害、暴击、生命、防御、共鸣效率、冷凝/热熔/导电/气动/衍射/湮灭伤害加成

---

## 配置说明

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `强化策略` | 下拉框 | 传统 / 渐进式 |
| `当前套装` | 下拉框 | 通用 + 17 套装 |
| `有效词条` | 多选框 | 13 个词条类型 |
| `X权重` | 数值滑块 | 13 个词条各一个，默认 1.0 |
| `最低得分≥` | 数值 | 评分模式/传统模式的最终阈值 |
| `Pause after Success` | 开关 | 成功后暂停，手动复核 |

配置保存在 `configs/` 文件夹，复制即可导出/导入。

## 运行环境

- **操作系统：** Windows
- **Python：** 3.12
- **游戏分辨率：** 16:9（最低 1600x900）
- **游戏语言：** 简体中文 / 繁体中文
- **帧率：** 稳定 60 FPS

## 从源码运行

```bash
pip install -r requirements.txt --upgrade
python main.py        # Release 版本
python main_debug.py  # Debug 版本（显示识别框等调试信息）
```

## 项目结构

```
ww-echo/
├── main.py / main_debug.py         # 入口
├── config.py                       # 全局配置
├── src/
│   ├── echo_stats.py               # 词条档位数据 + 评分工具
│   ├── echo_set_templates.py       # 17 套装预期词条模板
│   ├── Labels.py                   # UI 模板匹配标签
│   ├── globals.py                  # 全局状态
│   ├── scene/WWScene.py            # 场景状态缓存
│   └── task/
│       ├── EnhanceEchoTask.py      # ⭐ 批量强化声骸
│       ├── ChangeEchoTask.py       # 批量修改主属性
│       ├── BaseWWTask.py           # 任务基类
│       ├── WWOneTimeTask.py        # 一次性任务 mixin

```
